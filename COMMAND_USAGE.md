# Formix 命令行使用文档

> 适用版本：v1.2.1+
> 命令行页使用独立的 FFmpeg 引擎，与转换页面互不阻塞。

---

## 一、功能概述

Formix 命令行页是一个内置于软件中的 FFmpeg 工具链终端，支持：

- `ffmpeg` — 媒体转换
- `ffprobe` — 媒体信息查询
- `ffplay` — 媒体播放预览

它不是系统终端，仅允许执行 FFmpeg 工具链命令，不支持 `cmd`、`powershell`、`bash` 等。

### v1.2.1 新特性

- **独立引擎**：命令行拥有独立的 FFmpeg 处理实例。执行命令时视频/音频/图片等转换页面不受影响，反之亦然，两者可同时进行。
- **半透明终端**：终端背景为毛玻璃半透明效果，可透过终端看到应用背景。
- **动态输出优化**：GPU 编码器状态行（如 `86.41 M-A: 0.000 fd=0 aq=416KB...`）自动过滤，不会刷屏。

## 二、安全限制

为避免误操作，以下内容会被拦截：

| 拦截项 | 说明 |
|--------|------|
| `\|` | 管道 |
| `&` | 后台/串联 |
| `;` | 命令分隔 |
| `>` `<` | 重定向 |
| `` ` `` | 命令替换 |
| `$()` | 命令替换 |

> **不支持**：多条命令串联、管道重定向、系统命令。这些限制是设计意图，不会解除。

## 三、基础操作

### 进入页面

切换到"命令行"标签页，终端自动聚焦，等待输入。

### 按键说明

| 按键 | 功能 |
|------|------|
| `Enter` | 执行当前命令 |
| `Ctrl + Enter` | 换行（输入多行命令） |
| `Ctrl + C` | 中断正在运行的命令 |
| `↑` | 上一条历史命令 |
| `↓` | 下一条历史命令 |

### 交互说明

- 命令运行中按 `Enter` 不会重复提交。
- 鼠标拖动可选择日志文本并复制。
- 底部显示进度条（仅 `ffmpeg` 转换任务）。

## 四、路径规则

### 推荐写法

含空格的路径始终加**英文双引号**：

```bash
ffmpeg -i "D:/Videos/input file.mp4" "D:/Output/output.mp4"
```

### 无空格路径

可以直接写，不需要引号：

```bash
ffprobe D:/Videos/input.mp4
```

### 路径位置

- 输入文件：通过 `-i` 指定
- 输出文件：命令末尾最后一个非选项参数

从稳定性考虑，建议统一使用带引号路径。

## 五、常用命令示例

### 格式转换

```bash
# 转 MP4（H.264 + AAC）
ffmpeg -i "input.mkv" -c:v libx264 -c:a aac "output.mp4"

# 转 WebM（VP9 + Opus）
ffmpeg -i "input.mp4" -c:v libvpx-vp9 -c:a libopus "output.webm"

# 流复制（超快，不重编码）
ffmpeg -i "input.mp4" -c copy "output.mkv"
```

### 音频处理

```bash
# 提取音频为 MP3
ffmpeg -i "input.mp4" -vn -c:a libmp3lame "output.mp3"

# 提取无损音频
ffmpeg -i "input.mp4" -vn -c:a flac "output.flac"
```

### 信息查询

```bash
# 基本信息
ffprobe "input.mp4"

# 完整流信息
ffprobe -hide_banner -show_format -show_streams "input.mp4"

# JSON 格式输出
ffprobe -v quiet -print_format json -show_format -show_streams "input.mp4"
```

### 媒体播放

```bash
ffplay "input.mp4"
```

### 帮助

```bash
ffmpeg -h
ffprobe -h
ffplay -h
```

## 六、进度与中断

### 进度条

运行 `ffmpeg` 转换命令时，终端底部显示实时进度条。

- `ffplay` 和 `ffprobe` 不显示转换进度条
- GPU 编码器状态行自动原地更新，不刷屏

### 中断命令

按 `Ctrl + C` 向当前命令发送终止信号。

---

## 七、历史记录

命令行页面会记录本次会话中执行过的命令：

- `↑` 回填上一条命令
- `↓` 回填下一条命令
- 连续重复的命令不会重复记录
- 命令运行期间历史切换不生效

---

## 八、与转换页面的关系

命令行页使用独立的 FFmpeg 引擎，与视频/音频/图片/M3U8/音视频转换页面完全解耦：

- 命令行执行中，转换页面可正常使用，反之亦然
- 无需等待对方完成
- 各自的取消操作互不影响

---

## 九、组件缺失

如果缺少以下可执行文件，终端会直接提示：

- `ffmpeg` 缺失
- `ffprobe` 缺失
- `ffplay` 缺失

请到**设置 > FFmpeg** 下载或更新 FFmpeg 套件。

## 十、常见问题

### 命令被拒绝

通常原因：
1. 命令不以 `ffmpeg`/`ffplay`/`ffprobe` 开头
2. 包含被拦截的 shell 控制符（`|`、`&`、`;` 等）
3. 引号不成对，解析失败

### 路径打不开

检查清单：
1. 路径中是否有空格——加英文双引号
2. 输入和输出路径是否写反
3. 文件扩展名与编码参数是否匹配

### ffplay / ffprobe 不能运行

本地缺少对应可执行文件，到设置页重新下载 FFmpeg 套件。

### 为什么不能像系统终端一样

命令行页的设计目标是安全执行 FFmpeg 工具链命令，不是系统 Shell。
如需系统终端操作，请使用操作系统自带的终端。
